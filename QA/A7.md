D0 — integration guard obligatoire

Avant la validation élargie, ajoute un helper unique :

def derive_audit_surface(
    repo_path: Path,
    changed_files: list[str],
    *,
    mode: Literal["changed", "impact"] = "impact",
    max_files: int = 50,
) -> list[str]:
    ...

Règles :

mode="changed":
  retourne changed_files uniquement

mode="impact":
  retourne [entry.path for entry in build_impact_cone(...)]

Ne modifie pas AuditRequest.scope.changed_files.
changed_files reste le diff brut.
L’audit surface devient une surface dérivée.

Ensuite :

normalize:
  audit_surface = derive_audit_surface(repo, changed_files, mode="impact")
  obligations = extract_obligations(repo, audit_surface)

score-trace --request:
  utiliser la même audit_surface pour borner U(t)

Le point important est de distinguer clairement :

diff_scope = fichiers réellement modifiés
audit_surface = diff_scope ∪ impact_cone borné

Ne mélange pas les deux dans le schema public sans l’indiquer.
