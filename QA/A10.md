E0.2 — separate layer vs vendored core

Ma recommandation : separate layer first.

Ne modifie pas le core OIDA vendored maintenant.

Pourquoi :

1. Le core OIDA reste la référence formelle.
2. Le graphe est déjà utile pour repair/audit.
3. La fusion graph-aware n’est pas encore calibrée.
4. Une couche séparée permet de tester sans casser la signification de V_net.
5. On pourra proposer un core-change plus tard si la couche séparée devient stable.

Architecture proposée :

vendored OIDA core
  -> computes official local OIDA quantities when inputs are trusted

fusion_readiness layer
  -> decides if official fusion is allowed

experimental_fusion layer
  -> optional, clearly marked shadow/diagnostic

Donc E0 doit trancher :

official_summary:
  stays null

readiness_report:
  populated

experimental_shadow_summary:
  allowed only if explicitly non-authoritative
