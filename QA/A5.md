Le problème actuel est bien identifié : PreconditionSpec ne contient que name, weight, verified, et _preconditions_for() retourne aujourd’hui une seule précondition avec tout le poids parent et verified = obligation.status == "closed". C’est exactement ce qu’on doit corriger maintenant.

Point critique : ne ferme pas l’obligation parent avant de l’expanser.

Aujourd’hui, _link_evidence_to_obligations() peut transformer une obligation open en closed, puis _preconditions_for() convertit cette fermeture grossière en une précondition unique vérifiée. Si on garde ce flux, on perd toute possibilité de grounding partiel. Il faut donc inverser la logique :

Avant:
  evidence -> obligation.status = closed -> one PreconditionSpec verified=True

Après:
  obligation + evidence -> N child PreconditionSpec
  each child verified independently
  grounding = weighted fraction of verified children

Concrètement, je veux que tu remplaces la logique par quelque chose de ce type :

def _preconditions_for(
    obligation: Obligation,
    tool_evidence: list[ToolEvidence] | None = None,
    changed_files: list[str] | None = None,
) -> list[PreconditionSpec]:
    ...

Ou bien avec un petit EvidenceView interne si c’est plus propre.

Règle absolue :

sum(child.weight for child in children) == obligation.weight

Utilise un helper pour éviter les erreurs de flottants :

def _weighted_children(
    obligation: Obligation,
    parts: list[tuple[str, float, bool]],
) -> list[PreconditionSpec]:
    ...

Et teste avec pytest.approx.

ADR-20 — Obligation is not isomorphic with PreconditionSpec

Decision:
  An Obligation is a testable commitment.
  A PreconditionSpec is one atomic support condition consumed by OIDA grounding.
  One obligation may expand to N preconditions.
  Child weights must conserve parent weight.
  Evidence is evaluated per child, not blindly inherited from parent status.

Rejected:
  Keeping one obligation = one precondition.
  Auto-closing parent obligations before expansion.
  Treating pytest-green as proof of all subconditions.
