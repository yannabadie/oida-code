# QA/A54 - ADR-73 / G-6d.3 corpus pinning tranche

Date: 2026-04-30

## cgpro decision

cgpro selected `ADR-73 / G-6d.3 corpus pinning tranche` as the next
autonomous block after ADR-72.

Binding scope:

- Use only existing records from `reports/calibration_seed/index.json`
  whose `partition` is still `null`.
- Pin exactly 4 records.
- Assign exactly 3 records to `train` and exactly 1 record to
  `holdout`.
- Move the live corpus from N=14 to N=18: 13 train + 5 holdout.
- Keep G-6d open because the documented target remains N>=20.
- Freeze partitions before scoped pytest feasibility, CI outcome
  inspection, replay inspection, or provider judgement.
- Use `label_source=ai_authored_public_diff_review`,
  `human_review_required=true`, and `llm_assist_used=true` for all
  four new pins.

Forbidden scope:

- No fresh GitHub harvesting or PAT GitHub path.
- No provider calls as evidence.
- No replay generation, round-trip output, verifier JSON, or grounded
  report.
- No runtime changes under `src/oida_code`.
- No provider import, provider tool-calling runtime path, MCP runtime,
  gateway default change, GitHub Action change, clone-helper flag
  widening, or official fusion-field emission.
- No product-verdict wording, G-6d closure claim, broad
  generalisation claim, predictive-validity claim, product-safety
  claim, or future replay correctness claim.

Partition rule:

1. Sort selected `case_id` values lexicographically.
2. Compute `sha256('g6d3-holdout:' + case_id)`.
3. Assign the lowest hash to `holdout`.
4. Assign the other three selected cases to `train`.

Required artifacts:

- `reports/phase6_d_3_pinning/selection.{json,md}`
- `reports/phase6_d_3_pinning/feasibility.{json,md}`
- `reports/calibration_seed/index.json`
- `tests/test_phase6_d_3_pinning.py`
- Historical adjustment for ADR-72 live-count guards if needed
- Backlog, project status, expansion protocol, progress, and decision
  log updates

Stop conditions:

- Fewer than four existing unpinned records pass metadata plus local
  public-diff checklist.
- A selected candidate needs provider judgement, private data, PR
  comments, fresh GitHub API harvesting, or a new clone-helper flag.
- A selected candidate cannot be reduced to one narrow behavior claim
  and one narrow runnable pytest scope.
- Any post-freeze feasibility run fails or flakes.
- Any artifact implies G-6d is closed at N=18.

## Stop outcome

The first post-freeze feasibility attempt stopped before scoped pytest
outcome was reached.

Frozen attempted set:

- `seed_058_pallets_itsdangerous_378` - intended train
- `seed_071_simonw_sqlite_utils_689` - intended train
- `seed_074_simonw_sqlite_utils_658` - intended train
- `seed_159_hynek_structlog_759` - intended holdout

Failure mode:

- `clone_target_at_sha.py` cloned and installed
  `pallets/itsdangerous@7f4dcf83...`.
- The attempted `--install-group tests` step failed because the target
  has no PEP 735 `[dependency-groups]` table.
- The target's own `tox.ini` uses `deps = -r requirements/tests.txt`,
  and `requirements/tests.txt` contains `pytest==8.1.1`.
- cgpro ruled that manually installing `requirements/tests.txt` would
  widen the dependency-install path inside this block because seed_058
  had already been documented as the older requirements-pattern risk.

Decision after cgpro stop consultation:

- Do not manually install `requirements/tests.txt`.
- Do not replace `seed_058` after freeze.
- Do not commit G-6d.3 as a successful N=18 corpus advance.
- Revert `reports/calibration_seed/index.json` to the ADR-72 live
  corpus state.
- Commit only the honest stop artifacts and guards.
