Petit E1.0 avant la formule shadow

Avant d’écrire la shadow fusion, corrige ou clarifie un point de sémantique dans fusion_readiness.

Aujourd’hui, si des préconditions réelles existent mais qu’aucune n’est vérifiée, grounding peut être classé comme default. Conceptuellement, ce n’est pas idéal : un grounding à 0 avec de vraies préconditions non vérifiées est un vrai signal négatif, pas un default structurel.

Je veux donc un petit patch E1.0 :

E1.0 — grounding zero semantics

If total_preconditions > 0:
  grounding status should be "real" even when verified = 0

If total_preconditions == 0:
  grounding status should be "missing"

Do not confuse:
  no evidence found
  vs
  no precondition model exists

Tests attendus :

test_grounding_zero_with_real_preconditions_is_real_not_default
test_grounding_missing_when_no_preconditions
test_grounding_partial_remains_real

Ce patch ne débloque pas V_net. Il rend seulement le signal plus honnête pour la shadow fusion.

Formule expérimentale minimale

Je propose une shadow fusion conservatrice, uniquement en pression de risque :

base_pressure(event) =
  w_grounding   * (1 - grounding)
+ w_static      * static_failure_pressure
+ w_trajectory  * trajectory_pressure
+ w_completion  * (1 - completion)

Avec :

w_grounding  = 0.40
w_static     = 0.20
w_trajectory = 0.25
w_completion = 0.15

Puis propagation graphe :

constitutive_pressure(child) =
  max(
    base_pressure(child),
    max(parent_pressure * edge.confidence * alpha_constitutive)
  )

supportive_audit_pressure(child) =
  max(
    base_audit_pressure(child),
    max(parent_pressure * edge.confidence * alpha_supportive)
  )

Paramètres :

alpha_constitutive = 0.80
alpha_supportive   = 0.40

Règle clé :

constitutive edge => may increase shadow_debt_pressure
supportive edge   => increases audit pressure, not automatic invalidation

C’est cohérent avec Block C : constitutive sert à rouvrir, supportive sert à auditer.

Invariants obligatoires

E1 doit tester les invariants avant de regarder les chiffres :

1. authoritative is always false.
2. readiness != official_ready means official summary fields stay null.
3. Adding a failing constitutive parent cannot reduce child pressure.
4. Adding a supportive parent cannot force reopen semantics.
5. Removing all graph edges degrades to local-only shadow pressure.
6. Cycles do not create infinite propagation.
7. Propagation is deterministic.
8. Graph presence alone does not make blocked readiness official.
9. corrupt_plausible_success remains diagnostic, not public corrupt_success.

Pour les cycles, utilise une propagation bornée :

max iterations = min(10, number_of_events + 1)
operator = max()
values clipped to [0, 1]

Le max() rend la propagation monotone et idempotente. Pas de somme cumulative, sinon une boucle import/test peut gonfler artificiellement la pression.
