# QA/A56 - ADR-75 Dependency Policy

Date: 2026-04-30

## cgpro consultation

Thread: `repo-product-vision-review`

Conversation: `69f329be-0dd4-838f-8687-d68190f21e7d`

## Decision requested

After ADR-74, G-6d could not resume until the repo recorded a policy
for candidates whose scoped tests require historical
`requirements/*.txt` / `tox.ini` dependency patterns.

cgpro was asked to choose between:

- policy-only reject/defer before freeze;
- adding structural requirements-file support to the clone helper;
- a narrower alternative.

## cgpro decision

cgpro selected policy A:

> requirements-file / tox-only test-dependency candidates are rejected
> or deferred before partition freeze for G-6d.4.

The rationale was boundary preservation. ADR-73 already showed that
post-freeze rescue would widen the dependency-install boundary, and
ADR-66 pins the current predeclared clone-helper flag set. Adding
support now would be a new manual-lane capability, not a prerequisite
for the next clean G-6d tranche.

## Accepted scope

- Update the G-6d expansion protocol with an explicit ADR-75 policy.
- Update product strategy, project status, backlog, and agent handoff.
- Add a policy report and regression tests.

## Forbidden scope

- No `--install-requirements-file` flag.
- No `pip install -r requirements/tests.txt` support.
- No edit to `scripts/clone_target_at_sha.py`.
- No edit to `reports/calibration_seed/index.json`.
- No new G-6d.4 pinning artifacts.
- No replay bundles or `round_trip_outputs`.
- No runtime verifier, provider, gateway, MCP, or GitHub Action default
  changes.
- No official OIDA fusion-field unlock.

## Next step

After ADR-75 is committed, G-6d.4 may start candidate selection from
the existing `reports/calibration_seed/index.json` pool.

Any candidate requiring `requirements/*.txt`, tox `deps = -r ...`,
manual `pip install -r ...`, or a new requirements-file clone-helper
flag must be rejected or deferred before freeze.
