## What this verdict directs

### Phase 6.1'g — narrow `--install-extras` corrective (per next_action)

Scope:

* Edit `scripts/clone_target_at_sha.py` only.
* Add `--install-extras EXTRAS` flag (probably repeatable, or
  comma-separated).
* The flag changes the install invocation from
  `pip install -e <path>` to `pip install -e "<path>[EXTRAS]"`.
* Extend `_import_smoke_check` (or add a sibling function) to
  also verify `python -m pytest --version` when extras are
  requested — fail fast with a clear `target_bootstrap_gap`
  banner if pytest is not in the venv after the extras-install.
* Hermetic tests (monkeypatch + importlib pattern from Phase
  6.1'f) covering: extras-flag forwarding, smoke-check
  execution order, refusal/failure reporting, MANUAL_EGRESS_SCRIPT
  preservation.
* Docstring update with extras-syntax examples.

NOT in scope:

* Verifier edits, generator edits, replay-prompt edits.
* Build-backend specific flag plumbing.
* Forensic-archive expansion.

### Fresh freeze-rule holdout pass (next-next commit)

After Phase 6.1'g lands and tests are green:

* Frozen manifest at the post-6.1'g SHA.
* Re-run seed_065 + seed_157 with `--install-extras tests` +
  `--import-smoke <target>` + `python -m pytest --version`

  `diagnostic_only` from a real test failure → archive as
